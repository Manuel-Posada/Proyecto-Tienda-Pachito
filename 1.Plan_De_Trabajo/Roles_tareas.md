## 🔹 Descripción de Roles

**Product Owner:**

- Define y prioriza el Product Backlog  
- Representa las necesidades del negocio  
- Asegura que el equipo desarrolle valor para el proyecto  

**Scrum Master:**

- Facilita la metodología Scrum  
- Elimina impedimentos del equipo  
- Asegura el cumplimiento de las buenas prácticas ágiles  

**Development Team:**

- Desarrolla las funcionalidades del sistema  
- Implementa tanto el frontend como el backend  
- Colabora en la toma de decisiones técnicas  

**Stakeholder:**

- Aporta requerimientos y expectativas del sistema  
- Evalúa el valor entregado por el proyecto

## 🔹 Organización del Trabajo

El equipo trabajará bajo la metodología **Scrum**, donde:

- El **Product Owner** define qué se debe construir  
- El **Scrum Master** facilita el proceso  
- El **Development Team** se encarga de la implementación  
- El **Stakeholder** valida que el producto cumpla con las expectativas  

Esto permite una mejor comunicación, organización y entrega continua de valor en el proyecto.

---

## 🎯 Justificación

La asignación clara de roles y tareas permite:

- Evitar confusiones dentro del equipo  
- Mejorar la eficiencia en el desarrollo  
- Asegurar el cumplimiento de objetivos  
- Facilitar la gestión del proyecto  

De esta manera, cada integrante conoce sus responsabilidades y contribuye al éxito del sistema de gestión de la tienda Pachito.
